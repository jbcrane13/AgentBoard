iPhone Tab Structure

The issue proposes 4 tabs. Here are layout options for organizing the content.

A

4 Tabs (as proposed)

Board · Chat · Sessions · More

More tab contains: Settings, Notes, History, Agents, Milestones, Epics. Board tab has project picker + Kanban columns. Sessions tab has active sessions + session logs.

Pros

  • Clean, focused tabs
  • Chat gets dedicated space
  • Matches issue spec

Cons

  • "More" tab is a grab bag (6+ items)
  • Agents/Epics buried
B

5 Tabs

Board · Chat · Sessions · Agents · More

Promotes Agents to a top-level tab since it's a primary workflow. More tab slims down to: Settings, Notes, History, Milestones, Epics.

Pros

  • Agent monitoring is one tap away
  • 5 tabs is still clean on iPhone

Cons

  • Slightly more crowded tab bar
  • More tab still has 5 items
C

4 Tabs — Board absorbs more

Board · Chat · Agents · Settings

Board tab becomes the hub: project picker, Kanban, plus a segmented control to switch between Board/Epics/Milestones/Ready Queue. Sessions are accessed from Board context (tap a session badge). Agents tab shows sessions + agent workload. Settings replaces "More".

Pros

  • No "More" dumping ground
  • Every tab has clear purpose
  • Board is the natural home for project views

Cons

  • Board tab is heavier
  • Sessions less discoverable
